文件名称：setmap.m
输入参数：地图的长、宽、障碍数目
输出参数：地图，障碍所在位置索引
作用：	设定地图大小和障碍数目，随机生成输入数目的障碍点存放到输出中的数组里。


文件名称：extend_obs.m
输入参数：地图、障碍数目、障碍索引、功能号、扩展大小
输出参数：障碍膨胀后地图，膨胀障碍所在位置索引
作用：	障碍膨胀，输出障碍为全黑，即扩展障碍得大小；
	扩展大小：左右同时扩展，即2最后扩展为1+2+2=5；

文件名称：fdis.m
输入参数：坐标点1，坐标点2，权重，功能号
输出参数：距离
作用：	1：曼哈顿距离
	2：欧式距离
	3：对角距离

文件名称：AS.m
输入参数：含有障碍得地图，起点坐标，终点坐标
输出参数：规划出得路径，运行次数。
作用：	传统A*算法，使用欧式距离，8个方向扩散，步长为1。
	没有做多次转弯得惩罚，故会出现多次转弯得情况（后续需要优化一下）。


文件名称：SAS.m
输入参数：含有障碍得地图，起点坐标，终点坐标
输出参数：规划出得路径，运行次数。
作用：	稀疏A*算法，使用欧式距离，起步12个方向扩散，之后5个方向扩散，步长可通过run修改参数。
	目前是12个起飞方向，后续可能会规定起飞方向；
	当步长固定得时候，降落点必须要满足固定步长会出现路径过长得情况：
解决方法：如果终点规定了降落角度得时候，从降落得角度进行扩展5个方向，得到可以进入降落点得坐标，使这些坐标作为伪终点，即到达这些坐标后，就可以认为路径规划成功。

输出参数：规划出得路径
